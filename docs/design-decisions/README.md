# Design Decisions

Record product and UI design choices for Limon: what was decided, why, and what was rejected.

## How to add a decision

Create one markdown file per decision, named `YYYY-MM-DD-short-title.md`.

Suggested structure:

```md
# Title

**Date:** YYYY-MM-DD  
**Status:** Proposed | Accepted | Superseded

## Context

What problem or question prompted this decision.

## Decision

What we chose.

## Alternatives considered

What else was evaluated and why it was not chosen.

## Consequences

What this locks in, and what to watch for later.
```
